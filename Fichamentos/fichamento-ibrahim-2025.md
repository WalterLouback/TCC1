# Using NLP and AI to Enhance Software Documentation and Code Comprehension

Ibrahim, A.; Khan, S.; Ali, S. "Using NLP and AI to Enhance Software Documentation and Code Comprehension," International Journal of Software Engineering and Applications, vol. 14, no. 1, pp. 1-18, 2025. doi: [10.63163/jpehss.v3i2.292](https://doi.org/10.63163/jpehss.v3i2.292)

## 1. Fichamento de Conteúdo

Este estudo explora a aplicação de técnicas de Inteligência Artificial e Processamento de Linguagem Natural para gerar automaticamente documentação precisa e contextualmente relevante para código de software. Utilizando modelos baseados em transformers como CodeT5, GraphCodeBERT e GPT-3, os autores desenvolveram e avaliaram um sistema capaz de produzir resumos significativos de funções e classes de código. A metodologia incluiu análise comparativa entre documentação escrita por humanos e gerada por IA, utilizando métricas quantitativas (BLEU, ROUGE, F1) e feedback qualitativo de desenvolvedores profissionais. Os resultados indicam que a documentação gerada por IA melhora significativamente a legibilidade do código e a eficiência dos desenvolvedores, reduzindo o tempo de compreensão e aumentando a precisão no entendimento de código complexo. A integração em tempo real da ferramenta dentro de ambientes de desenvolvimento provou ser benéfica para suporte contínuo de documentação. Apesar dos desafios em lidar com código específico de domínio e interpretar segmentos mal escritos, o impacto geral na qualidade da documentação é substancial. O trabalho enfatiza o potencial de ferramentas orientadas por NLP para automatizar e padronizar práticas de documentação, oferecendo uma solução escalável para um dos desafios persistentes da engenharia de software.

## 2. Fichamento Bibliográfico

* _Transformer-Based Models_ (modelos baseados em transformers) são arquiteturas de aprendizado profundo que utilizam mecanismos de atenção para processar sequências, incluindo CodeT5, GraphCodeBERT e GPT-3 (seção de metodologia).
* _Code Comprehension_ (compreensão de código) refere-se à capacidade de desenvolvedores entenderem a funcionalidade, estrutura e propósito de código de software (introdução).
* _NLP Techniques_ (técnicas de NLP) são métodos computacionais para processar e analisar linguagem natural, aplicados aqui para geração de documentação (contexto teórico).
* _Real-Time Integration_ (integração em tempo real) permite que ferramentas de documentação funcionem continuamente dentro de IDEs durante o desenvolvimento (resultados).
* _Domain-Specific Code_ (código específico de domínio) refere-se a implementações especializadas que requerem conhecimento contextual profundo para documentação adequada (discussão de limitações).

## 3. Fichamento de Citações

* _"While AI still faces challenges in handling domain-specific code and interpreting poorly written segments, the overall impact on documentation quality is substantial."_
* _"This research underscores the potential of NLP-driven tools to automate and standardize documentation practices, offering a scalable solution to one of software engineering's longstanding challenges."_
